# 深圳地铁厕所地图

一个面向深圳地铁出行场景的轻量厕所查询地图。当前仓库提供静态 Web MVP，可按线路、站名、出口、位置和服务设施筛选站点厕所信息。

> 当前版本用于验证产品形态和数据结构，厕所点位仍是示例数据。正式发布前需要用官方查询结果或实地核验数据替换。

## 项目状态

- 类型：静态网页
- 构建：无构建步骤
- 地图：Leaflet + OpenStreetMap 瓦片
- 数据：本地 JSON 文件
- 当前数据：示例数据，非完整官方数据

## 功能

- 地图点位浏览
- 支持换乘站的多线路筛选
- 站名、出口、位置关键词搜索
- 仅看站内厕所、无障碍厕所、母婴室
- 站点详情侧栏
- 数据与界面分离，后续可替换为完整官方数据

## 运行

由于浏览器对本地 JSON 读取有限制，建议启动本地静态服务：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

如果 8000 端口被占用，可以换一个端口：

```bash
python3 -m http.server 8080
```

## 数据说明

厕所数据位于 `data/toilets.json`。当前数据是用于验证产品形态的示例集，不代表完整、实时、官方结果。正式发布前应以深圳地铁官方车厢二维码查询结果、深圳地铁/交通运输局公开信息或人工实地核验为准。

已知公开信息显示，深圳地铁已上线全网卫生间信息二维码查询功能；后续可以把该官方查询结果整理为结构化数据后替换本仓库示例数据。

参考：

- 深圳市交通运输局：深圳地铁全面上线二维码查询卫生间服务 https://jtys.sz.gov.cn/zwgk/jtzx/cxtx/content/post_12139317.html
- 深圳市市场监督管理局：《地铁厕所设计及管理规范》解读 https://amr.sz.gov.cn/attachment/1/1524/1524247/11901082.pdf

## 数据字段

```json
{
  "id": "station id",
  "name": "站名",
  "lines": ["线路"],
  "lat": 22.0,
  "lng": 114.0,
  "paidArea": true,
  "accessible": true,
  "babyCare": false,
  "location": "厕所位置描述",
  "exit": "临近出口",
  "note": "补充说明",
  "updatedAt": "2026-06-23"
}
```

字段说明：

- `id`：唯一标识，建议使用稳定英文短横线命名。
- `name`：站点中文名。
- `lines`：站点所属线路数组。换乘站写入多条线路，例如 `["1号线", "3号线"]`。
- `lat` / `lng`：站点或厕所附近坐标，用于地图定位。
- `paidArea`：是否位于付费区内。`true` 表示站内/付费区，`false` 表示非付费区或出入口通道。
- `accessible`：是否有无障碍厕所或无障碍设施。
- `babyCare`：是否有母婴室或母婴相关设施。
- `location`：厕所具体位置描述，尽量包含楼层、方向、通道或参照物。
- `exit`：临近出口或方向。
- `note`：补充说明、核验限制或特殊提示。
- `updatedAt`：该条数据最后核验日期，格式为 `YYYY-MM-DD`。

## 数据维护流程

1. 收集官方二维码查询结果、深圳地铁公开页面、深圳市交通运输局公开信息或实地核验记录。
2. 将每个厕所点位整理为 `data/toilets.json` 中的一条记录。
3. 对换乘站保留同一站点记录，并在 `lines` 中填写所有相关线路。
4. 更新 `updatedAt`，在 `note` 中记录无法确定的信息。
5. 校验 JSON：

```bash
python3 -m json.tool data/toilets.json >/dev/null
```

6. 本地启动服务并检查搜索、筛选和地图标记是否正常。

## 贡献建议

提交数据时，请尽量提供以下信息：

- 站名和线路
- 具体位置，例如站厅层、站台层、某出口方向、换乘通道附近
- 是否在付费区
- 是否支持无障碍厕所
- 是否有母婴室
- 信息来源或核验日期

不要提交未经确认的“完整数据”。如果来源不确定，请在 `note` 中明确标注。
